# Plan: Telegram Cron Management (자연어 기반)

## 현재 상태

### CLI (완전 지원)
- `cclaw cron add <bot>` - 인터랙티브 생성 (name, schedule/at, timezone, message, skills, model)
- `cclaw cron remove <bot> <job>` - 삭제
- `cclaw cron enable <bot> <job>` / `disable` - 활성/비활성
- `cclaw cron run <bot> <job>` - 즉시 실행

### Telegram (읽기 + 실행만)
- `/cron list` - 목록 조회
- `/cron run <name>` - 즉시 실행

### 기존 CRUD 함수 (cron.py)
- `add_cron_job(bot_name, job)` - 이미 존재
- `remove_cron_job(bot_name, job_name)` - 이미 존재
- `enable_cron_job(bot_name, job_name)` - 이미 존재
- `disable_cron_job(bot_name, job_name)` - 이미 존재

## 목표

Telegram에서 자연어로 cron job을 생성할 수 있도록 `/cron` 핸들러 확장

## 핵심 설계: 자연어 → Claude 파싱

### `/cron add <자연어>`

사용자가 **어떤 언어로든** 자연어로 스케줄과 작업을 한 줄로 입력하면, Claude가 파싱하여 cron job을 생성한다. 언어에 국한되지 않는다.

```
/cron add 매일 아침 9시에 이메일 요약해줘
/cron add every morning at 9am summarize emails
/cron add 平日の午後6時にニュースをまとめて
/cron add 30분 뒤에 회의 알림
/cron add in 30 minutes remind me about the meeting
/cron add 내일 오후 2시에 보고서 확인
/cron add next Tuesday at 2pm check the report
/cron add 매주 월요일 10시에 주간 리포트 작성
/cron add every Monday at 10am write weekly report
```

### 파싱 흐름

```
사용자 입력 ("/cron add 매일 아침 9시에 이메일 요약해줘")
    ↓
claude -p 호출 (one-shot, 파싱 전용 프롬프트)
    ↓
JSON 응답 파싱
    {
      "type": "recurring",
      "schedule": "0 9 * * *",
      "message": "이메일 요약해줘",
      "name": "email-summary"
    }
    ↓
add_cron_job() 호출
    ↓
확인 메시지 전송
```

### 파싱 프롬프트 설계

Claude에게 보내는 one-shot 프롬프트:

```
You are a cron schedule parser. Parse the following natural language input into a cron job definition.
The input can be in ANY language (Korean, English, Japanese, Chinese, etc.).

Current local time: {current_time}
Timezone: {timezone}

Rules:
- Determine if this is "recurring" (repeating schedule) or "oneshot" (one-time execution)
- For recurring: generate a standard cron expression (5 fields: minute hour day month weekday)
- For oneshot: generate an ISO 8601 datetime string in the given timezone
- Extract the task message (the part that describes WHAT to do, not WHEN)
  - Keep the message in the ORIGINAL language of the input
- Generate a short kebab-case name (English, 2-3 words) from the task message

Respond ONLY with JSON, no other text:
{
  "type": "recurring" | "oneshot",
  "schedule": "cron expression" (recurring only),
  "at": "ISO datetime" (oneshot only),
  "message": "task description in original language",
  "name": "short-kebab-name-in-english"
}

Input: {user_input}
```

### 타임존 결정

1. GLOBAL_MEMORY.md에서 timezone 읽기 (예: `Asia/Seoul`)
2. 없으면 시스템 로컬 타임존 (`tzlocal` 또는 `time.tzname`)
3. 최종 fallback: UTC
4. 파싱된 결과의 job에 timezone 필드 자동 설정

→ 사용자가 "아침 9시"라고 하면 로컬 타임존 기준 9시로 해석됨

### 응답 예시

```
⏰ Cron job created:

  Name: email-summary
  Schedule: 0 9 * * * (Asia/Seoul)
  Message: 이메일 요약해줘
  Next run: 03-06 09:00
```

One-shot 예시:
```
⏰ Cron job created:

  Name: meeting-reminder
  Run at: 2026-03-05 15:30 (Asia/Seoul)
  Message: 회의 알림
```

## 구현 범위

### 1. cron.py 변경

- `parse_natural_language_schedule()` 함수 추가
  - `claude -p`를 one-shot으로 호출하여 자연어 → JSON 파싱
  - JSON 파싱 실패 시 에러 반환
  - 타임존 결정 로직 포함
- `resolve_default_timezone()` 함수 추가
  - GLOBAL_MEMORY.md → 시스템 로컬 → UTC 순서

### 2. handlers.py 변경

- `cron_handler`의 subcommand 분기에 `add`, `remove`, `enable`, `disable` 추가
- `add`: 자연어 텍스트 → `parse_natural_language_schedule()` → `add_cron_job()`
- `remove/enable/disable`: 기존 CRUD 함수 직접 호출

### 3. 테스트 (코드 테스트, mock 기반)

Claude 응답을 mock하여 코드 흐름을 검증한다. 일반 `pytest`로 실행.

- `test_cron.py`:
  - `parse_natural_language_schedule()` — JSON 파싱 성공 → job dict 반환
  - `parse_natural_language_schedule()` — JSON 파싱 실패 → 에러 반환
  - `parse_natural_language_schedule()` — recurring type → schedule 필드 존재
  - `parse_natural_language_schedule()` — oneshot type → at 필드 존재
  - `resolve_default_timezone()` — GLOBAL_MEMORY에서 timezone 읽기
  - `resolve_default_timezone()` — fallback 순서 (global memory → 시스템 로컬 → UTC)
  - job name 충돌 시 suffix 추가
- `test_handlers.py`:
  - `/cron add` — 성공 시 확인 메시지

  - `/cron remove` — 성공/실패 메시지
  - `/cron enable` / `/cron disable` — 상태 변경 확인

### 4. 파싱 품질 평가 (수동, 실제 Claude 호출)

프롬프트 변경 시 수동으로 실행하여 다국어 자연어 파싱 품질을 검증한다.
`pytest -m evaluation`으로 분리하여 일반 CI에서는 제외.

```
tests/evaluation/test_cron_parsing_quality.py
```

**평가 케이스:**

```python
EVALUATION_CASES = [
    # --- Recurring (Korean) ---
    ("매일 아침 9시에 이메일 요약해줘", "recurring", "0 9 * * *"),
    ("평일 오후 6시에 뉴스 정리", "recurring", "0 18 * * 1-5"),
    ("매주 월요일 10시에 주간 리포트", "recurring", "0 10 * * 1"),
    ("매월 1일 오전 10시에 월간 보고", "recurring", "0 10 1 * *"),

    # --- Recurring (English) ---
    ("every morning at 9am summarize emails", "recurring", "0 9 * * *"),
    ("every weekday at 6pm news summary", "recurring", "0 18 * * 1-5"),
    ("every Monday at 10am weekly report", "recurring", "0 10 * * 1"),
    ("every 1st of the month at 10am monthly report", "recurring", "0 10 1 * *"),

    # --- Recurring (Japanese) ---
    ("毎日朝9時にメールをまとめて", "recurring", "0 9 * * *"),
    ("平日の午後6時にニュースまとめ", "recurring", "0 18 * * 1-5"),

    # --- Oneshot (Korean) ---
    ("30분 뒤에 회의 알림", "oneshot", None),
    ("내일 오후 2시에 보고서 확인", "oneshot", None),
    ("다음 주 화요일 3시에 미팅 준비", "oneshot", None),

    # --- Oneshot (English) ---
    ("in 30 minutes remind me about the meeting", "oneshot", None),
    ("tomorrow at 2pm check the report", "oneshot", None),
    ("next Tuesday at 3pm prepare for meeting", "oneshot", None),
]
```

**검증 기준:**

| 항목 | 기준 | 허용 오차 |
|------|------|-----------|
| `type` (recurring/oneshot) | 정확도 100% | 불허 |
| `schedule` (cron expression) | recurring 케이스 정확 일치 | `0 9 * * *` 와 `0 9 * * 0-6` 같은 동치 표현은 허용 |
| `message` 언어 보존 | 입력 언어와 동일 | 불허 (번역하면 안 됨) |
| `name` 형식 | 영어 kebab-case | 불허 |
| `at` (ISO datetime) | oneshot에만 존재, 유효한 ISO format | 시간값 자체는 검증 안 함 (현재 시간 의존) |

## 고려사항

### Claude 파싱 비용
- 파싱용 one-shot 호출은 haiku 모델 사용 (빠르고 저렴)
- 프롬프트가 짧아서 토큰 소비 최소

### 파싱 실패 처리
- JSON 파싱 실패 시: "스케줄을 인식하지 못했습니다. 예시: `/cron add 매일 아침 9시에 이메일 요약해줘`"
- Claude 호출 타임아웃 시: 일반 에러 메시지

### job name 충돌
- 같은 이름이 이미 존재하면 `-2`, `-3` suffix 추가
